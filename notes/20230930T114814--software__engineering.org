* What define the /Software Engineering/ (from now on SW)?
The code that you wrote for yourself, with no docs or engineering is called /program/, is intended for private use.
The /software/ on the other hand is well documented, before any line of code is written there is a process of engineering and so on.
* Ethics in Software Engineering
+ confidentiality
+ expertise
+ intellectual property
+ don't abuse your skill
+ More [[https://www.acm.org/about-acm][here]]
* What define a good Software
+ Easy of maintainance and evolution
+ Meets the clients requirements
+ Good docs
+ Good engineering (this is redundant)
+ Respect the ethics
* One model of SW for all projects
Each software projects have some qualities and peculiarity and such perks require differents models of SW to produce a good software.
