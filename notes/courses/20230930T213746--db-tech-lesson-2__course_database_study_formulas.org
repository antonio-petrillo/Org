* Secondary memory
Although tertiary memory is larger, in term of data that can be written into it, it is to slow for our study case, we will focus on secondary memory.
** HDD
An HDD is divided into:
1. *Tracks*
2. Each tracks is divide into *Sectors*
3. Each sectors is furtherly divided into *Blocks*
* Records
Our ~Records~ (relational tables) can be written to files.
The records will be written into HDD blocks.
There are few cases:
+ A record fit perfectly into a block
+ A record fit /not/ perfectly into a block (1 or more)
+ A record doesn't fit into a block (we should *SPAN* our block over multile blocks)
* Blocking factor
Consider the following:
$$Block_{size} = B$$
$$Record_{size} = R$$
We define the *blocking factor* as:
$$bfr = \lfloor\frac B R \rfloor$$
* Operation
When we study how to write the records we should consider the following operations:
*Record at a time*:
+ Open, Close, Reset
+ Find, FindNext
+ Read
*Set at a time*:
+ Insert, Delete, Modify
+ Find All, Find n, FindOrdered
+ Reorganize
