#+title: Use Case Diagram
#+filetags: :architecture:engineering:uml:

* What is it used for
To model the interaction between the system and external actor (stakeholders or external softwares).
It is also used to describe how different actor would use the system.
* Summary
| Symbol          | Role     | Description                                  |
|-----------------+----------+----------------------------------------------|
| stickman        | actor    | actor (stakeholders or external entity)      |
| ellipses + text | action   | operation performed by the system            |
| arrow           | relation | specify which actor can perform which action |
* Note
Since this type of diagram is extremely simple it usually completed with a table that explains all the actors, actions and relations.
